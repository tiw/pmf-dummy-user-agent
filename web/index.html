THE FIVE-LAYER FRAMEWORK

      
五层设计框架

      
生成符合场景的虚拟用户，不是让 LLM "假装"成一个人，而是让 LLM 以特定身份和场景约束来运行。


      

        

          
01

          
角色定义层

          
结构化身份卡片：demographics、psychographics、behavioral traits。每个字段都影响 LLM 的注意力分布。

        

        

          
02

          
大模型条件化

          
用 System Prompt 把 persona 转化为运行时约束。用具体约束替代模糊描述。

        

        

          
03

          
场景注入层

          
同样的 persona，在不同场景下行为完全不同。定义时间压力、参与动机、初始态度。

        

        

          
04

          
记忆系统层

          
跨会话状态追踪：trust_level、emotional_state、exposure_count。渐进式认知演化。

        

        

          
05

          
行为引擎层

          
注意力过滤器 + 响应生成器 + 一致性守卫。控制虚拟用户如何"反应"。
